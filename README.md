# KnowledgeBase AI

A production-quality RAG (Retrieval-Augmented Generation) knowledge assistant that lets you upload documents or ingest web pages, then chat with an AI that answers questions using only your ingested content вҖ” with inline source citations for every answer.

Built with **Next.js 14**, **LangChain.js**, **OpenAI**, and **Pinecone**.

## Local Setup

### Prerequisites

- Node.js 20+
- OpenAI API key
- Pinecone account and index

### 1. Clone and install

```bash
git clone <your-repo-url> knowledgebase-ai
cd knowledgebase-ai
npm install --legacy-peer-deps
```

### 2. Create a Pinecone index

1. Sign up at [pinecone.io](https://www.pinecone.io/)
2. Create a **serverless** index with:
   - **Name:** `knowledgebase` (or set `PINECONE_INDEX_NAME`)
   - **Dimensions:** `1536` (matches `text-embedding-3-small`)
   - **Metric:** `cosine`
3. Copy your API key from the Pinecone console

### 3. Configure environment variables

```bash
cp .env.local.example .env.local
```

Edit `.env.local`:

```env
OPENAI_API_KEY=your_openai_api_key_here
PINECONE_API_KEY=your_pinecone_api_key_here
PINECONE_INDEX_NAME=knowledgebase
```

### 4. Run the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## How It Works

The RAG pipeline has two main flows:

### Ingestion

1. **Upload** вҖ” User drops a PDF, TXT, or DOCX file, or pastes a public URL
2. **Parse** вҖ” Raw text is extracted using the appropriate parser
3. **Chunk** вҖ” Text is split into 800-token chunks with 100-token overlap (LangChain `RecursiveCharacterTextSplitter`)
4. **Embed** вҖ” Each chunk is embedded with OpenAI `text-embedding-3-small`
5. **Store** вҖ” Vectors + metadata are saved in Pinecone

### Query

1. **Embed** вҖ” The user's question is embedded with the same model
2. **Retrieve** вҖ” Pinecone returns the top 5 most similar chunks (cosine similarity)
3. **Assemble** вҖ” Retrieved chunks are formatted into a context string
4. **Generate** вҖ” GPT-4o-mini streams an answer constrained to the context, with source citations
5. **Display** вҖ” The UI shows the streamed answer with expandable source cards
